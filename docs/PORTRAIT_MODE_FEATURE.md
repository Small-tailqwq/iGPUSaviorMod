# 竖屏优化功能说明

## 功能概述

本功能为 iGPU Savior Mod 新增了竖屏优化支持，当检测到分辨率高度大于宽度（竖屏模式）时，自动调整主相机参数以优化竖屏显示效果。

## 功能特性

### 自动检测与切换
- **实时检测**：在 `Update()` 中每帧检测分辨率变化
- **智能判断**：当 `Screen.height > Screen.width` 时识别为竖屏模式
- **自动切换**：进入/离开竖屏时自动应用/恢复相机参数

### 相机参数调整

#### 1. 位置调整（Z轴前移）
```csharp
newPosition.z += 1.5f; // 沿Z方向靠近角色
```
- 缩小裁切范围
- 使角色更接近相机视野中心

#### 2. 俯角调整（Pitch）
```csharp
currentRotation.x += 10f; // 增加10度俯角
```
- 向下俯视
- 使角色上半身更好地居中显示

#### 3. FOV/正交尺寸调整
```csharp
// 透视相机
cam.fieldOfView += 8f;

// 正交相机
cam.orthographicSize *= 1.15f;
```
- 扩大视野范围
- 填充竖屏画面

### 参数保存与恢复

#### 原始参数缓存
```csharp
private Vector3 originalCameraPosition;
private Quaternion originalCameraRotation;
private float originalCameraFOV;
private float originalCameraOrthoSize;
private bool hasOriginalCameraParams;
```

#### 智能恢复机制
- 首次进入竖屏时保存原始参数
- 离开竖屏时完整恢复所有参数
- 确保多次切换不会累积误差

## 镜像模式兼容性

### 设计原则
- **独立调整**：竖屏优化仅修改相机 transform 和 FOV
- **不修改场景**：不缩放 GameObject，避免与镜像模式冲突
- **分离逻辑**：竖屏检测独立于镜像模式检测

### 兼容性验证
- ✅ 竖屏优化 + 镜像模式可同时启用
- ✅ 相机参数调整互不干扰
- ✅ 镜像翻转基于 RenderTexture，与相机位置无关

## 性能考虑

### 检测开销
- **每帧检测**：简单的整数比较（`Screen.height > Screen.width`）
- **性能影响**：可忽略不计（纳秒级）
- **优化策略**：仅在状态变化时执行参数调整

### 内存占用
- 额外变量：5个（2个bool + 3个Transform参数）
- 内存开销：< 100 bytes

## 使用示例

### 场景1：手机竖屏游戏
```
原始分辨率：1920x1080（横屏）
切换至：1080x1920（竖屏）
→ 自动检测并应用竖屏优化
→ 相机前移、俯角增加、FOV扩大
```

### 场景2：回到横屏
```
从竖屏切换回：1920x1080（横屏）
→ 自动检测并恢复原始参数
→ 相机位置、旋转、FOV完全恢复
```

## 可调参数

如需调整竖屏优化效果，可修改 `ApplyPortraitCameraAdjustment()` 方法中的以下参数：

| 参数 | 当前值 | 说明 |
|------|--------|------|
| Z轴前移距离 | `1.5f` | 调整相机与角色的距离 |
| 俯角增量 | `10f` | 调整向下俯视角度 |
| FOV增量 | `8f` | 透视相机视野扩大幅度 |
| OrthoSize倍率 | `1.15f` | 正交相机尺寸放大倍率 |
